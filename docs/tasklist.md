## Итерация 16 — Полировка и документирование

**Цель:** довести поведение агента до продакшен-качества и зафиксировать в документации.

- [ ] System prompt: границы компетенции (только услуги компании, вежливый отказ), тон, инструкции по инструментам

- [ ] LangSmith: один родительский трейс на весь диалог (сообщение → ответ), включая tools и embeddings
- [ ] LangSmith: трассировка по потокам (`thread_id` = user_id + uniq; новый thread при `/start`)

**Проверка:** три сценария в Telegram:

В LangSmith — родительский трейс на каждый сценарий, отдельные потоки после `/start`.
