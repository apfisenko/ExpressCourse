## Итерация 1 — Каркас и Telegram

- [ ] Структура проекта и зависимости
- [ ] Конфигурация из `.env`
- [ ] Скрипты `install` / `run`

## Итерация 2 — Ответ через LLM

**Проверка:** задать вопрос — получить осмысленный ответ модели (без памяти между сообщениями).

## Итерация 3 — История диалога

**Проверка:** ссылаться на предыдущие реплики в одной сессии — бот учитывает контекст.

## Итерация 4 — Роль ассистента

- [ ] Системный промпт с описанием роли
- [ ] Ответы соответствуют заданному поведению и стилю

**Проверка:** задать вопросы по теме роли — ответы в нужном тоне и фокусе.

---

## Итерация 5 — Ошибки и устойчивость

- [ ] Обработка сбоев LLM и Telegram

**Проверка:** при недоступности API бот не падает, а сообщает о проблеме.
